# timedog

### Зачем это сделано

Классический **timedog** отвечает на вопрос «что изменилось между двумя снимками Time Machine» **текстом в терминале**. Для короткого списка этого достаточно, но для большого бэкапа неудобно: **не видно структуру каталогов**, трудно **ориентироваться** в дереве и **открыть файл**, чтобы понять, что именно поменялось внутри.

**`timedog-server` как раз для другого сценария:** **веб-интерфейс**, где изменения показаны **деревом**, можно **ходить по папкам**, открывать файлы и смотреть **дифы** (текст и hex) между **старым** и **новым** снимком на диске. **Навигация по файлам и просмотр отличий** — главная фишка стека на Go/React; формат отчёта JSONL и HTTP API обслуживают те же правила сравнения для сканов, сохранённых отчётов и скриптов. Perl-скрипт остаётся простым вариантом для командной строки.

timedog — Perl-скрипт, который показывает, какие файлы изменились между снимками Time Machine (размеры до/после, сводка, опционально сводка по глубине `-d`, сортировка, фильтры). **В этом репозитории также есть `timedog-server`**: программа на Go со встроенным React UI, которая реализует **те же правила сравнения** (обход двух каталогов снимков с учётом inode и hard-link) и выдаёт отчёт в **JSONL**, а не только вывод в терминал.

## Что мы переписали или добавили (по сравнению с классическим Perl)

| Область | Суть |
|--------|------|
| **Движок сравнения** | Новая реализация на Go (`internal/scan`): обход корня *нового* снимка, сравнение каждого пути с *старым* через `lstat` (inode, размеры), отсечение неизменившихся поддеревьев через `SkipDir`, как в Perl. |
| **Параллельный обход** | По желанию обход через [`fastwalk`](https://github.com/charlievieth/fastwalk) (`fast_walk` в API и в UI); остаётся режим последовательного `filepath.WalkDir`. |
| **Формат отчёта** | JSONL: первая строка — метаданные (`timedog-report-meta`), далее по одному JSON на изменённый путь. По расширению `.jsonl.gz` — gzip. |
| **Стриминг при скане** | Фоновый скан **сразу создаёт файл отчёта**, пишет **meta**, затем **по строке на каждый изменённый путь по мере обхода**, после завершения — **пересортировка / rollup `-d`** и **полная перезапись файла** итоговым отчётом (totals, skipped, финальный порядок). |
| **HTTP API + UI** | `timedog-server`: список снимков (`tmutil`), скан с прогрессом по SSE, дерево отчёта, diff текста/hex для файлов внутри заданных корней. Статика Vite/React вшивается в бинарник. |
| **Вспомогательное** | `timecopy.py`, `timediff`, `UsingTimecopy.md` — назначение прежнее; подробности в соответствующих файлах. |

**Perl-скрипт** (`./timedog`) остаётся в репозитории для работы из терминала и как ориентир; сервер ориентирован на совпадение *модели* с timedog, а не на побайтовое совпадение формата вывода в консоли.

## Классический Perl-скрипт (терминал)

1. Терминал (`/Applications/Utilities`).
2. `/path/to/timedog -h` — справка в пейджере (`q` для выхода).
3. Пример: `/path/to/timedog -d 5 -l` — сводка до 5 уровней сегментов пути, без симлинков.

Можно передать метку бэкапа аргументом; `-t` выводит список снимков. Пример вывода:

```shell
$ ~/Desktop/timedog -d 5 -l
==> Comparing TM backup 2009-01-15-080533 to 2009-01-15-070632
    1.6KB->    2.9KB        /.Backup.log
...
==> Total Backup: 111 changed files/directories, 8.08MB
```

Числа в квадратных скобках (например `[26]`) при `-d` — сколько вложенных изменений под каталогом.

## timedog-server (Go + веб UI)

Бинарник `timedog-server` отдаёт небольшой HTTP API и собранный React UI. Сравнивает два каталога снимков по той же логике, что и Perl timedog (inode / hard-link), пишет **JSONL-отчёт** (мета + объект на изменённый путь), **не включает содержимое файлов** в отчёт. Тело файла читается **только** при открытии панели Text/Hex и если корни снимков ещё доступны на диске.

### Требования

- **macOS** (`tmutil listbackups -m` и обход ФС).
- **[Go](https://go.dev/dl/)** для сборки сервера; **[Node.js](https://nodejs.org/)** — только если пересобираете веб UI для встраивания в бинарник.
- Смонтированный **том Time Machine** (или образ бэкапа) перед **новым сканом** или перед чтением содержимого файлов с этого тома.

### macOS: полный доступ к диску (важно)

Без **полного доступа к диску** (Full Disk Access) macOS часто **запрещает чтение** каталогов бэкапа, системных путей и т.п. — возможны пустые списки снимков, пропуски при обходе или ошибки «Operation not permitted».

1. Откройте **Системные настройки** (на старых версиях — **Защита и безопасность** / **Конфиденциальность**) → **Конфиденциальность и безопасность** → **Полный доступ к диску**.
2. Включите доступ для приложения, которое **запускает** `timedog-server`:
   - если запускаете из **Терминала** или **iTerm** — добавьте это приложение в список и включите переключатель;
   - если сервер стартует из IDE (Cursor, VS Code и т.д.) — добавьте IDE или запускайте бинарник из обычного терминала после выдачи прав терминалу.
3. После изменения списка **закройте и снова откройте** терминал (или IDE), чтобы права применились.
4. При необходимости добавьте сам бинарник `timedog-server`, если запускаете его из Finder или обёртки — должен быть разрешён процесс, который реально ходит по ФС.

По смыслу это то же, что и для Perl `timedog`, когда нужно читать защищённые области.

### Сборка

Из корня репозитория:

```shell
go build -o timedog-server ./cmd/timedog-server
```

Чтобы встроить продакшен UI в бинарник, соберите фронтенд и скопируйте файлы в `cmd/timedog-server/web/dist`:

```shell
cd web && npm install && npm run build
# затем скопируйте web/dist/* в cmd/timedog-server/web/dist
```

Либо укажите при запуске `-static /путь/к/dist`, чтобы не копировать статику в репозиторий.

### Запуск

```shell
./timedog-server -addr 127.0.0.1:8080
```

Откройте в браузере: **http://127.0.0.1:8080** (или тот хост/порт, что передали в `-addr`).

**Разработка (UI с перезагрузкой):** в одном терминале — `./timedog-server`, в другом — `cd web && npm run dev`. Dev-сервер Vite проксирует `/api` на Go; открывайте URL, который покажет Vite (часто `http://localhost:5173`).

### Поведение

- **Скан:** выполняется в фоне на сервере; закрытие вкладки не отменяет задачу — только кнопка **Отменить**. После завершения выдаётся session id для дерева и контента.
- **Отчёт:** `.jsonl` или `.jsonl.gz`; загрузка/парсер распознаёт gzip по сигнатуре.
- **Безопасность:** эндпоинты контента резолвят пути только внутри `old_root` / `new_root` сессии.
